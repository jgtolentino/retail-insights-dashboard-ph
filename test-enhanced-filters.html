üß™ Test Scenarios

1. URL Synchronization

Navigate to your dashboard and:

  1. Change some filters (date range, category, brand, etc.)
  2. Check the URL bar - it should update automatically
  3. Copy the URL and open it in a new tab
  4. ‚úÖ Filters should be restored from the URL
Example: http://localhost:5173/consumer-insights?start=2025-04-01&end=2025-04-30&category=Cigarettes&brand=Marlboro&dayType=weekend

2. LocalStorage Persistence

  1. Set some filters on any dashboard page
  2. Close the browser tab completely
  3. Open the dashboard again
  4. ‚úÖ Your last filter selections should be restored

To verify in DevTools: localStorage.getItem('retail-dashboard-filters')

3. Browser Navigation

  1. Change filters multiple times
  2. Use browser Back button
  3. ‚úÖ Previous filter states should be restored
  4. Use browser Forward button
  5. ‚úÖ Next filter states should be restored

4. Debouncing

  1. Open Network tab in DevTools
  2. Rapidly change date range multiple times
  3. ‚úÖ API calls should be debounced (only final change triggers request)
  4. Look for "Updating filters..." indicator during debounce

5. Page-Specific Filters

Navigate between pages and observe:

  • Transaction Trends: Only Date Range and Location should be active
  • Product Mix: Date Range, Category, Brand, Location should be active
  • Consumer Insights: All filters should be active

‚úÖ Irrelevant filters should show with reduced opacity and info tooltips

6. Reset Functionality

  1. Set multiple filters
  2. Click "Reset All" button
  3. ‚úÖ All filters should return to defaults
  4. ‚úÖ URL should be cleared of filter parameters
  5. ‚úÖ LocalStorage should be cleared
